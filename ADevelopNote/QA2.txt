16. belongsTo 跟 belongsToMany 差異為何? 需解釋到資料表結構。
    public function category()
    {
        return $this->belongsTo(Category::class);
    }
    public function tags(): BelongsToMany
    {
        return $this->belongsToMany(Tag::class)->withTimestamps();
    }

    Ans:
    belongsTo用於一對多關係，代表「我屬於某個東西」，ex:[文章] belongsTo [分類]，透過foreignKey查找
    belongsToMany用於多對多關係，代表「我共同屬於某些東西」，需透過pivot table

    16-1. 使用這兩個時，資料表的命名是否有限制?
        假如A belongsTo B, A中的foreign key命名預設為"B_id"
        假如A belongsToMany B, 需透過pivot table "A_B_table"/"B_A_table"
            而pivot table中的id也嚴格遵守"{model}_id"的規定

19. 如果有要檔案上傳，應該要設定檔案的路徑處理
說明 config/filesystems.php 中如何處理。
    Ans:
    $path = $request->file('avatar')->store('avatars', 'public');
    會在'public'磁碟的根目錄建立'avatars'資料夾，並將檔案存儲其中
    存取時透過設定的象徵連結找到 ('links' => [...]) (php artisan storage:link)

    19-1.圖檔名稱怎麼辦? 在public根目錄又怎麼會跟storage有關? 所以是建立在storage中，還是public? 兩者差異?
        Ans:
        public為磁碟名，是laravel filesystem中的抽象概念，代表存放可公開存取的檔案
        "php artisan storage:link"會建立虛擬連結
        讓"storage/{filename}"的請求指向"storage/app/public/{filename}"的實際檔案位置
        透過<form>上傳的圖檔在 storage/ 中會採用隨機生成的檔名，避免重複問題

22. ArticleRequest 已經有驗證了，建立與更新時還需要驗證嗎? 兩個驗證差異為何?
    Ans:
    ArticleRequest檢查有登入session，即[是否為本人操作該帳戶]
    建立/更新時的驗證為[該帳戶是否有對該文章執行操作的權限]

    22-1. 我問的是表單驗證，不是權限
    return [
        'title' => ['required','string','max:150'],
        'body' => ['required','string','min:10'],
        // 'published_at' => ['nullable','date'],
        'tags' => ['nullable','array'],
        'tags.*' => ['integer','exists:tags,id'],
    ];
    與
    $validated = $request->validate([
        'title' => 'required|string|max:255',
        'body' => 'required|string',
        'category_id' => 'exists:categories,id',
        'tags' => 'array',
        'tags.*' => 'exists:tags,id',
    ]);

        Ans:
        Request->validate() 是用來驗證輸入資料是否符合規則(長度, 型別, 必要性等)
        如果驗證失敗，Laravel 會自動導回前頁並帶有錯誤訊息（HTTP 422）
        驗證成功則將request資料傳入$validated中
